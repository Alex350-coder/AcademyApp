# Cambios — 2026-06-23

## 1. Fix: Dos Columnas no se Alineaban (Derecha más pequeña)

**Problema:** La columna derecha del ReportsPage (CoursePerformanceChart) quedaba más corta que la izquierda porque solo mostraba 8 filas fijas y dejaba un espacio vacío considerable abajo.

**Causa raíz:** El cálculo de `itemsPerPage` en el `ResizeObserver` no consideraba el gap de 16px (`space-y-4`) entre los items. Usaba `Math.floor(h / 52)` cuando debía ser `(h + gap) / (52 + gap)`. Esto producía un `itemsPerPage` incorrecto, dejando espacio sobrante o recortando contenido.

Además, el estado inicial `useState(8)` mostraba un número arbitrario hasta que el ResizeObserver se disparaba, causando un flash visual.

**Solución:**
- Agregada constante `GAP = 16` que refleja el `space-y-4` entre items
- Fórmula corregida: `(h + GAP) / (ITEM_HEIGHT + GAP)` donde cada item ocupa 52px de contenido + 16px de gap = **68px**
- `useLayoutEffect` mide sincrónicamente antes del primer paint para evitar flash
- Estado inicial cambiado de `8` a `1` (se corrige inmediatamente con la medición sincrónica)
- Función extraída `calcItemsPerPage(h)` para usar la misma lógica en ambos efectos

**Archivos modificados:**
- `frontend/src/features/dashboard/director/components/CoursePerformanceChart.tsx:9-10` — nuevas constantes `GAP` y fórmula corregida
- `frontend/src/features/dashboard/director/components/CoursePerformanceChart.tsx:23` — `useState(1)` en vez de `useState(8)`
- `frontend/src/features/dashboard/director/components/CoursePerformanceChart.tsx:33-50` — nuevo `useLayoutEffect` + `ResizeObserver` con fórmula correcta

## 2. Fix: JWT Refresh Perdía Roles (Páginas de gestión no cargaban)

**Problema:** Los endpoints de gestión (Teachers, Students, Courses, Classrooms) devolvían 403 después de 15 minutos. El flujo era:
1. Access token expira (15 min)
2. Frontend hace refresh → obtiene nuevo token **sin roles**
3. El nuevo token no tiene `@PreAuthorize` válido → 403
4. Frontend entra en bucle refresh-logout

**Causa:** `AuthController.java:96` llamaba `createAccessToken(userId, null, null)` en vez de propagar los claims del token anterior.

**Solución:** El refresh ahora extrae `email` y `roles` del token expirado y los pasa al nuevo access token.

**Archivos modificados:**
- `backend/.../identity/infrastructure/controller/AuthController.java:96` — `createAccessToken(userId, email, roles)` en vez de `createAccessToken(userId, null, null)`
